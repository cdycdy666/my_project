## 卡片

### NVC-01 先区分观察和评价

- 来源：非暴力沟通
- 适用：对话里出现判断、定性、归因，表达容易让对方防御。
- 观察线索：总是、从不、你就是、你不在乎、快速下结论。
- 教练动作：把评价改写成可验证事实，再表达自己的感受和请求。
- 句式：刚才我看到/听到的是……我有点……我想确认……
- 避免：不要把自己的推断当作对方动机。
- 示例：
  - 反例：你根本不在乎这个群有没有用。
  - 更好：刚才我看到你只回了“好的宝宝”，我有点拿不准你是不是真的理解这个机器人要做什么，我想确认一下你的感受。
  - 为什么有效：先说可验证事实，再说自己的不确定，不把“没展开回复”直接解释成“不在乎”。

### NVC-02 把情绪翻译成需要，再提出具体请求

- 来源：非暴力沟通
- 适用：用户有情绪但表达停在抱怨，或需要对方配合但请求不清楚。
- 观察线索：烦、无语、算了、随便、你看着办。
- 教练动作：识别感受背后的需要，并给出一个对方能执行的小请求。
- 句式：我有点……因为我需要……你愿不愿意……？
- 避免：不要只说感受，不说希望对方做什么。
- 示例：
  - 反例：你怎么又不说话，我真的很烦。
  - 更好：我有点着急，因为我需要知道你真实的舒服程度。你愿不愿意直接告诉我：这个机器人现在是有帮助，还是有点打扰？
  - 为什么有效：把情绪落到“需要确认感受”，再提出一个对方可以马上回答的问题。

### NVC-03 请求要给对方选择空间

- 来源：非暴力沟通
- 适用：提出建议、安排、要求对方改变行为，尤其是用户先下指令后撤回。
- 观察线索：你把……、你应该……、算了不……也行、随便你。
- 教练动作：如果不是硬性要求，就把偏好和选择权同时说清楚。
- 句式：我的偏好是……但你也可以……你觉得哪个更方便？
- 避免：不要先像命令一样表达，再用补丁撤回。
- 示例：
  - 反例：你把它移出去吧。算了不移也行，随便你。
  - 更好：我的偏好是先把它留在群里试两天；如果你觉得被打扰，我们今晚就移出去。你觉得哪个更舒服？
  - 为什么有效：同时表达偏好和选项，避免让对方猜哪一句才是最终意思。

### NVC-04 先确认对方的感受和需要，再推进事情

- 来源：非暴力沟通
- 适用：对方回应短、弱、含糊，且关系沟通多于任务沟通。
- 观察线索：好的、嗯、都行、表情回应、没有展开。
- 教练动作：不要立刻进入下一步安排，先给一个轻量确认。
- 句式：听起来你是……我理解对吗？
- 避免：不要把简短同意理解成充分对齐。
- 示例：
  - 反例：好的，那我继续调。
  - 更好：听起来你是愿意先让我把它调好，但还不确定它会不会打扰正常聊天，我理解对吗？
  - 为什么有效：把对方的短回复当作需要确认的信号，而不是直接当作完全同意。

### VOSS-01 用标注降低对方防御

- 来源：掌控谈话
- 适用：对方可能有顾虑，用户想推进但不确定对方是否愿意。
- 观察线索：疑问句背后有担心、对方追问进度、对方没有直接拒绝。
- 教练动作：用“听起来/看起来/感觉上”标注可能状态，给对方修正空间。
- 句式：听起来你可能是在担心……是不是？
- 避免：不要直接断言对方心理。
- 示例：
  - 反例：你是不是觉得这个机器人很烦？
  - 更好：听起来你可能担心它会打扰我们正常聊天，是这样吗？
  - 为什么有效：用“可能”保留修正空间，降低对方被定性的感觉。

### VOSS-02 复述关键词，邀请对方多说一点

- 来源：掌控谈话
- 适用：对方说得很短，用户需要更多信息但不想逼问。
- 观察线索：一句短回应、一个关键词、含糊评价。
- 教练动作：重复对方最后几个关键字，并停顿等待。
- 句式：“调教好了吗？”
- 避免：不要连续追问多个问题。
- 示例：
  - 反例：还没，我刚才已经说过了，你等一下。
  - 更好：调教好了吗？你说的“调教好”，更在意它能记录，还是能给建议？
  - 为什么有效：先复述关键词表示听到了，再只补一个澄清问题，邀请对方展开。

### VOSS-03 用如何/什么类问题让对方参与设计方案

- 来源：掌控谈话
- 适用：需要共同决定，容易变成单方面安排。
- 观察线索：要不要移出、怎么设置、以后怎么聊、是否保留。
- 教练动作：把指令改为开放但有边界的问题。
- 句式：你希望我们怎么安排……会比较舒服？
- 避免：不要只给一个封闭选择，除非已经非常明确。
- 示例：
  - 反例：那以后就让它每天分析吧。
  - 更好：你希望它怎么参与会比较舒服？比如只晚上总结，还是我明确喊它时再分析？
  - 为什么有效：开放问题加上两个边界选项，让对方参与设计，而不是被动接受安排。

### VOSS-04 推进前做一句总结确认

- 来源：掌控谈话
- 适用：刚做出安排，对方短回复表示同意，下一步会改变群/工具/关系边界。
- 观察线索：好的、行、那就这样、我将保持。
- 教练动作：先总结双方共识，再执行动作。
- 句式：我确认一下：我们之后就在这个群聊，我保留……你那边……对吗？
- 避免：不要在共识未确认时快速执行多个动作。
- 示例：
  - 反例：行，那我现在就改。
  - 更好：我确认一下：先只记录这个群，晚上给一次建议；如果你觉得不舒服，我们再关掉。对吗？
  - 为什么有效：在改变工具设置前复述共识，降低误解和反复调整。

### LOCAL-01 请求要稳定，不要刚提出就撤回

- 来源：本项目沟通复盘经验
- 适用：用户连续补充、反转、降低要求，表达里出现不确定但没有说明原因。
- 观察线索：也行、算了、不移也行、随便。
- 教练动作：先想清楚偏好、底线和可选项，再一次性表达。
- 句式：我的建议是……如果你觉得……也可以……
- 避免：不要让对方猜哪一句才是最终版本。
- 示例：
  - 反例：要不把它拉进来？算了也不用，或者先试试？你看吧。
  - 更好：我的建议是先试三天，只让它做晚间总结；如果你觉得打扰，我们立刻停。你愿意这样试吗？
  - 为什么有效：把试验周期、范围和退出条件一次说清楚，减少对方的判断负担。
