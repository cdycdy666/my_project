# 视觉组件库

《实用思维工具》系列优先使用能表达“思考过程”的组件，而不是单纯堆文字。

## 核心组件

### C-TOOL-CARD

- 用途：介绍工具的一句话定义。
- 适合 scene：原则段。
- 风险：不要变成大字 PPT。

### C-MISTAKE-MAP

- 用途：展示观众默认误判。
- 适合 scene：hook、diagnosis。
- 风险：同屏不要超过 3 个误判。

### C-CAUSE-CHAIN

- 用途：展示“然后会怎样”的后果链。
- 适合工具：二阶思维、反向思考。
- 风险：链条不能太长，最多 4 节。

### C-TRADEOFF-SCALE

- 用途：展示选择和被放弃的东西。
- 适合工具：机会成本、取舍模型。
- 风险：不要做成财务图表。

### C-DECISION-LOG

- 用途：展示时间戳、假设、信号和复盘。
- 适合工具：决策日志。
- 风险：表格信息不超过 4 行。

### C-WHITEBOARD-GAP

- 用途：用空缺、划线、重写展示理解漏洞。
- 适合工具：费曼技巧。
- 风险：文字要有手写感，但必须清晰。

### C-PREMORTEM-REPORT

- 用途：展示“已经失败”的事故报告。
- 适合工具：事前验尸、反向思考。
- 风险：气氛不能太阴暗，重点是预防动作。

### C-ACTION-CHECKLIST

- 用途：结尾给今天或本周动作。
- 适合 scene：action、outro。
- 风险：动作不能超过 3 步。

## 视觉原则

- 一个 scene 只解决一个视觉任务。
- 每屏主信息只保留一个。
- 数字和步骤必须有动作对应。
- 系列统一使用“工具箱 / 白板 / 决策台 / 标注系统”的视觉隐喻。
