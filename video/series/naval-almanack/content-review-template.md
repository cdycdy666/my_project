# 内容优先评审模板

这个模板用于在视频制作前判断：这个选题值不值得做。

目标不是把《纳瓦尔宝典》讲全，而是每条视频只讲透一个能让观众停下来的判断。

硬规则：内容评分低于 9.0，不进入制作。

## 评审顺序

1. 选题是否有现实痛点
2. 观点是否足够锋利
3. 案例是否足够具体
4. 3 秒开头是否能抓人
5. 结尾是否有行动
6. 是否有平台钩子假设和封面误区
7. 是否能从“解释概念”升级为“解决一个当下焦虑”

## 评分维度

- 反常识程度：是否打破观众原来的默认想法
- 痛点强度：是否击中开发者、创作者、职场人的真实焦虑
- 案例清晰度：是否有一个能被想象出来的人和场景
- 观点锋利度：是否能压缩成一句有记忆点的话
- 行动可执行度：观众看完能不能当天做一件小事
- 口语化潜力：是否能被自然说出来，而不是只能写在文章里
- 平台钩子潜力：前 3 秒是否像平台语言，而不是书摘标题
- 封面成立度：第一眼是否能看到冲突，是否避开泛知识封面

## 进入制作条件

- 内容评分 >= 9.0
- 反常识程度、痛点强度、案例清晰度、观点锋利度至少 3 项为高
- 必须有一个具体人物和场景
- 必须有一句能被转述的记忆句

## 输出格式

```text
内容评分：
1-10 分。

是否进入制作：
是 / 否。

一句话观点：

现实冲突：

目标观众：

具体案例：

平台钩子假设：

封面误区：

记忆句：

最大亮点：

主要风险：

必须保留的内容：

下一步制作指令：
```
