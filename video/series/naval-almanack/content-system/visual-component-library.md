# 视觉组件库 v1

视觉组件库负责让系列形成稳定、可复用、可升级的画面语言。它不是限制创意，而是避免每条视频都临时拼 PPT。

## 使用原则

- 每个 scene 必须选择 1 个主组件，除非视觉导演说明为什么不用。
- 每条视频至少使用 3 个系列组件，其中至少 1 个必须承载案例场景。
- 优先使用低成本、可复用组件；高成本组件必须说明必要性。
- 组件服务叙事，不为装饰存在。

## 组件清单

### C-AUDIT-CARD 退休审计卡

- 适用：行动建议、结尾自查、具体清单。
- 输入：3 个待审计事项、1 个本周动作、1 句记忆句。
- 视觉：深色玻璃卡、编号圆点、底部高亮结论。
- 动效：编号依次点亮，最后一项用暖色描边。
- 成本：低。
- 风险：文字过多时像表格，必须每项不超过 14 个字。

### C-CALENDAR-DEBT 日历债务表

- 适用：时间被牺牲、周末被占用、会议过载。
- 输入：一周日历块、被占用时间、被拿走的个人时间。
- 视觉：日历网格 + 红/橙色债务块。
- 动效：债务块逐步覆盖空白时间。
- 成本：中。
- 风险：网格过密会看不清，手机端最多展示 5-7 个块。

### C-MESSAGE-PRESSURE 消息压力堆

- 适用：工作侵入生活、被迫响应、边界失守。
- 输入：3-5 条消息泡泡、时间戳、关键压力词。
- 视觉：聊天气泡从右侧压入，主角时间被挤压。
- 动效：气泡堆叠，背景轻微震动。
- 成本：低。
- 风险：不要模拟真实公司或个人隐私。

### C-TIME-BILL 时间账单

- 适用：把抽象时间成本变成可见账单。
- 输入：时间支出项、小时数、可复利事项。
- 视觉：收据 / 账单结构，低价值事项灰色，高价值事项金色。
- 动效：账单逐行打印，最终合计突出。
- 成本：低。
- 风险：数字必须来自脚本上下文，不能凭空出现。

### C-BEFORE-AFTER 双栏选择

- 适用：旧判断 vs 新判断、卖时间 vs 积累资产、被迫工作 vs 选择工作。
- 输入：左栏旧状态、右栏新状态、核心差异。
- 视觉：左右对照，左侧偏冷、右侧偏暖。
- 动效：左侧微暗，右侧逐步点亮。
- 成本：低。
- 风险：左右信息必须平衡，不能一边塞太多。

### C-DIAGNOSIS-STAMP 诊断盖章

- 适用：关键判断、转折句、问题定位。
- 输入：一句诊断、一个标签词。
- 视觉：档案页上盖章，标签如 `不是钱不够`、`是边界失守`。
- 动效：短促盖章，微抖，不要夸张。
- 成本：低。
- 风险：一条视频最多使用 2 次，避免廉价感。

### C-CHOICE-LOCK 选择权锁

- 适用：自由、选择权、被迫/自愿的区别。
- 输入：被锁住的选择、拿回的选择。
- 视觉：锁扣 / 开关 / 边界线。
- 动效：锁扣松开或边界线重画。
- 成本：中。
- 风险：不要做成励志海报，必须和具体案例绑定。

### C-ASSET-COMPOUND 资产复利图

- 适用：长期主义、杠杆、具体知识、作品积累。
- 输入：投入动作、沉淀资产、复利结果。
- 视觉：小点逐步连成资产曲线。
- 动效：前期慢，后期轻微上扬。
- 成本：中。
- 风险：不要像理财收益承诺，标注为能力/作品/信任复利。

### C-MEMORY-LINE 记忆句卡

- 适用：结尾、转折、封面候选。
- 输入：一句 14-24 字记忆句。
- 视觉：大字标题 + 小号来源标签 `NAVAL ALMANACK`。
- 动效：逐词显现，最后一个关键词高亮。
- 成本：低。
- 风险：不能堆副标题，留足呼吸感。

### C-COVER-CONTRAST 封面对比卡

- 适用：封面、发布首帧、章节标题。
- 输入：一句冲突标题、一个反常识副标题。
- 视觉：强对比排版，标题像编辑部封面。
- 动效：用于视频内时只做轻微推近。
- 成本：低。
- 风险：标题不能像营销号恐吓。

## 组件失败模式

- 组件好看但没有服务当句口播。
- 每段都用卡片，缺少场景证据。
- 动效太多，导致判断感变成广告感。
- 同屏文字超过观众静音阅读负荷。
- 用高成本组件解决低价值段落。

## 维护节奏

- 每条视频完成后，视觉导演和装配剪辑 Agent 共同标记哪些组件有效。
- 发布复盘如果发现封面或前 3 秒表现好，把对应组件加入“高表现组件”。
- 连续 3 条没有复用价值的组件，降级为实验组件。
