# 脚本实验室 v1

脚本实验室负责在正式 `script.md` 前完成结构试验。它的目标不是多写几版文案，而是在低成本阶段把钩子、案例、节奏和记忆句打磨到足够锋利。

## 必须先做实验的情况

- 新主题第一次制作。
- G0 内容评分刚过 9.0，但案例或钩子不够硬。
- 主题容易讲成书摘、鸡汤、理财课或课程广告。
- 上一条复盘指出前 3 秒、案例段或结尾行动不够强。

## 标准实验流程

1. 从内容原料库选 1-2 张原料卡。
2. 写 5 个开头钩子，覆盖不同平台语言。
3. 写 3 个案例角度，选择最能被画面呈现的一版。
4. 写 2 个反驳桥，提前处理观众可能不信的地方。
5. 写 3 句记忆句，选择最短、最能转述的一句。
6. 做 0-3s、3-15s、15-45s、45-75s、75-end 的留存检查。
7. 产出最终脚本策略，再进入 `script.md`。

## 钩子类型

| 类型 | 用法 | 示例方向 |
| --- | --- | --- |
| 反误判型 | 直接戳破默认想法 | 你可能并不是想退休。 |
| 诊断型 | 让观众意识到自己被说中 | 你不是不努力，是努力没有被放大。 |
| 代价型 | 告诉观众继续这样会损失什么 | 你正在把今天抵押给一个想象中的以后。 |
| 选择型 | 把抽象原则变成分叉 | 你是在卖时间，还是在积累资产？ |
| 审计型 | 让观众马上能自查 | 今天做一次退休审计。 |

## 案例角度选择标准

- 具体到人物、年龄或身份，但不要像小说人物设定。
- 必须有可视化动作：回消息、改方案、删会议、写清单、设置边界。
- 必须有代价：睡眠、周末、好奇心、作品时间、选择权。
- 不能只讲“她很焦虑”，要讲焦虑如何出现在日历、消息、账单或任务里。
- 时间推进必须自然。出现“三个月后”“六个月后”这类跳跃时，前文必须埋因果。

## 逐句专业检查

每一句口播写完后，脚本 Agent 必须逐句检查：

- 这句是否像真人会说，不像文章小标题。
- 这句是否只承担一个意思。
- 这句是否需要前一句铺垫。
- 这句是否引入了新时间、新数字、新人物或新动作。
- 这句是否能被视觉接住。
- 这句如果删掉，信息链是否断。

## 留存检查表

| 时间段 | 任务 | 必须避免 |
| --- | --- | --- |
| 0-3s | 抛出现实冲突或反误判 | 引用开头、解释背景、铺垫书名 |
| 3-15s | 建立新判断 | 连续抽象概念 |
| 15-45s | 用案例证明判断 | 案例无动作、无代价 |
| 45-75s | 给诊断和行动 | 行动太空、太像鸡汤 |
| 75-end | 留下记忆句和评论触发 | 口号化、强行总结 |

## 脚本红线

- 不能用书摘式开头：`纳瓦尔说过...`
- 不能只解释概念，没有现实代价。
- 不能连续三句抽象判断。
- 不能突然跳时间、跳数字、跳结论。
- 不能让行动建议停留在“你要思考”“你要长期主义”。
- 不能把结尾写成课程广告或励志口号。

## 评分标准

| 维度 | 分值 | 通过线 |
| --- | --- | --- |
| 前 3 秒停留 | 20 | >= 18 |
| 反直觉判断 | 15 | >= 13 |
| 案例具体性 | 20 | >= 18 |
| 口语自然度 | 15 | >= 13 |
| 留存推进 | 15 | >= 13 |
| 视觉可执行性 | 10 | >= 8 |
| 记忆句 | 5 | >= 4 |

总分低于 90，不进入正式 `script.md`。

## 输出要求

脚本 Agent 必须把实验结果写入每条视频的 `script-lab.md`，并在 `script.md` 中说明最终选择来自哪一组实验。
