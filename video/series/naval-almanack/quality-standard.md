# Naval Almanack 质量标准 v1

这个系列的目标不是“把《纳瓦尔宝典》做成书摘”，而是把每条原则做成一个能解决当下焦虑的短视频判断。

## 核心定位

- 观众感受：像听一个清醒、有经验的人，把一个复杂问题讲透。
- 内容气质：锋利、克制、具体，不鸡血、不带货、不像课程广告。
- 画面气质：深色编辑部档案风，像能力诊断报告，不像 PPT 知识卡片。
- 声音气质：冷静、可信、有一点私人感，不用夸张短剧腔或新闻播音腔。

## 不可妥协的质量线

- 内容评分低于 9.0：不进入制作。
- 没有内容原料卡：不进入制作。
- 开头 3 秒没有现实冲突：重写。
- 没有完成脚本实验室：不进入正式脚本。
- 没有具体人物和场景案例：重写。
- 只是在解释概念，没有解决一个真实焦虑：重写。
- 口播像书面文章，不能自然读出来：重写。
- 音频技术检查未通过：不进入最终成片。
- 发布前人工看听时判断配音明显不真实：不发布。
- 成片综合评分低于 9.0：不发布。
- 没有封面、发布标题和发布文案：不进入待发布。

## 内容标准

每条视频只讲一个判断，结构必须是：

1. 现实冲突：观众现在为什么会焦虑。
2. 反直觉判断：纳瓦尔这条原则如何改变默认想法。
3. 具体案例：一个能被想象出来的人、场景和选择。
4. 行动建议：观众今天或七天内能做什么。
5. 记忆句：一句能被转述的总结。

内容不追求讲全，追求讲透。宁可少讲一个要点，也不要牺牲锋利度和记忆点。

## 配音标准

- 固定方向：知识型旁白，冷静、有判断力、轻微私人感。
- 默认音色：Yunxi 作为当前基准，但每条都必须先做 30 秒试音。
- 文案句长：普通句 8-18 个字，关键句可以更短。
- 停顿要求：关键句前后 300-500ms，段落转场 150-250ms。
- 禁止风格：带货腔、短剧腔、新闻播音腔、过度情绪化鸡汤腔。
- 后期目标：人声清晰、响度稳定，背景音乐只托住情绪，不抢口播。

## 节奏标准

- 推荐时长：85-105 秒。超过 110 秒必须说明原因。
- 前 3 秒必须抛出冲突，不做铺垫。
- 每 8-12 秒必须有一个信息推进点。
- 每 20-30 秒必须有一次视觉状态变化。
- 结尾不喊口号，只给一个具体问题或动作。

## 视觉标准

- 必须保留系列标识：`NAVAL ALMANACK`、集数、主题英文。
- 必须保留进度感：顶部进度线或等价节奏提示。
- 字幕必须可静音观看，关键句必须烧录。
- 每条至少有一个“案例场景画面”，不能全是抽象卡片。
- 每条至少使用 3 个系列视觉组件，其中至少 1 个组件承载案例场景。
- 封面必须独立制作，不能只用视频截图凑数。

## 质量闸口

多 Agent 协作时，每个闸口都有明确责任人：

- G0：选题研究 Agent + 主编总控
- G1：内容主编 Agent + 脚本 Agent
- G2：配音导演 Agent
- G3：视觉导演 Agent
- G3.5：装配剪辑 Agent + 主编总控
- G4：视频评审 Agent + 主编总控
- G5：主编总控 + 发布复盘 Agent
- G6：主编总控 + 全体 Agent 输出记录

### G0 选题闸口

- 至少引用 1 张内容原料卡，或补写 `content-material-card.md`
- 原料卡必须包含当下焦虑、默认误判、案例种子和行动落点
- 内容评分 >= 9.0
- 反常识程度、痛点强度、案例清晰度、观点锋利度至少 3 项为高
- Notion `是否进入制作` 必须为是

### G1 文案闸口

- 有 `script-lab.md`
- `script-lab.md` 必须完成 5 个钩子、3 个案例角度、2 个反驳桥、3 句记忆句和留存检查
- `script-lab.md` 总分 >= 90 / 100
- 有 3 个开头钩子备选，并说明最终选择理由
- 有 `核心判断`、`观众旧判断 -> 新判断`、`情绪落点` 和 `不讲什么`
- 有清晰叙事骨架：hook、reframe、proof、diagnosis、action、memory line
- 有 1 个具体案例
- 有逐句口播稿
- 已完成逐句连续性自检
- 已完成口语自然度自检
- 有 `scene-contract.json`
- `scene-contract.json` 必须包含 `scene_energy`、`retention_goal`、`viewer_state_before`、`viewer_state_after` 和 `continuity_notes`
- 有屏幕文字和字幕重点
- 文案可以自然朗读，不像书面摘要
- 不允许无铺垫的时间跳跃、数字跳跃、结果跳跃或概念跳跃
- 每个新时间、新数字和新动作都必须有上下文来源或明确用途
- 脚本自评分 >= 90 / 100，且没有红线问题

### G2 配音闸口

- 先生成 30 秒试音
- 必须写入 `voice/voice-manifest.json`
- 必须给出 `technical_audio_pass`
- 必须标记 `audio_subjective_review`：`confirmed` 或 `pending`
- 如果系统没有实际听音频，不能把真实感预估写成已确认听感
- 如果试音技术状态或文本节奏有风险，优先改文案停顿，再考虑换音色

### G3 画面闸口

- 有 `visual-component-map.md`
- 每个 scene 必须选择主组件
- 至少 3 个组件来自 `content-system/visual-component-library.md`
- 至少 1 个组件承载案例场景
- 必须基于 `scene-contract.json` 写 `shot-list.md`
- 每个 scene 的主信息只能有 1 个
- 数字证据同屏不超过 3 组重点
- handoff 必须标注低/中/高制作成本，避免高成本画面成为系列负担
- `npx hyperframes lint` 0 errors / 0 warnings
- `npx hyperframes inspect` 0 layout issues
- `hyperframes/content.js` 和 `hyperframes/index.html` 不得残留旧主题文案、旧集数或旧视觉语义
- 抽取至少 4 个关键帧复看：开头、原则、案例、结尾
- 不允许标题断行难看、字幕遮挡主体、文字贴边

### G3.5 样片与装配闸口

- 必须产出 15-30 秒样片，或至少完成等价片段抽检
- 必须写 `assembly-check.md`
- 必须检查每个 scene 的切点、落字、停顿、转场、BGM ducking 风险
- 必须确认 `scene_energy` 和最终画面/配音节奏一致
- 必须标出“脚本没问题、视觉没问题、配音没问题，但拼在一起不顺”的风险
- G3.5 未通过前，不进入最终成片评审

### G4 成片闸口

- ffprobe 确认 1080x1920、音画时长匹配
- 视频评审评分 >= 9.0
- review 必须分为结构审和成片审
- `technical_audio_pass` 必须为通过
- 如果没有实际听音频，`audio_subjective_review` 必须标记为 `pending`，不能标记为已听审通过
- 必改建议全部处理

### G5 发布闸口

- 成片、无声版、字幕、配音、封面全部归档
- 发布标题、封面文案、发布文案、标签全部完成
- Notion `视频台账` 更新到最新版本
- Notion 视频页面已同步 `content-material-card.md`、`content-review.md`、`brief.md`、`script-lab.md`、`script.md`、`scene-contract.json`、`visual-component-map.md`、`shot-list.md`
- 已生成 `notion-sync-receipt.md`
- 发布清单保留“发布前人工完整看听一遍”，但该项不阻塞自动生产流程

### G6 Agent 复盘与自动迭代闸口

- 每条视频完成后必须生成 `agent-retrospective.md`
- 复盘必须覆盖各 Agent 的输入、输出、评分、问题和下一轮建议
- 只有发现高确定性、可复用、能降低返工或提升质量的改进点，才更新 `agent-roles/`、`multi-agent-sop.md` 或本质量标准
- 如果没有发现值得固化的改进，必须明确记录“本轮无高确定性迭代项，保持现有定义”
- 自动迭代不得为了形式感强行修改 Agent 定义
- 修改后的 Agent 定义或 SOP 必须同步到 Notion 后才算完成 G6
- 必须清理不再需要的子 Agent；如果保留任何 Agent，必须记录保留原因

## 当前标杆

当前三条 v4 共同构成系列质量基准：

- `纳瓦尔宝典-具体知识` v4：9.2 / 10，标杆在内容冲突和 AI 时代语境。
- `纳瓦尔宝典-长期主义复利原则` v4：9.1 / 10，标杆在把抽象长期主义转成“别把努力放进短期游戏”的传播钩子。
- `纳瓦尔宝典-杠杆` v4：9.1 / 10，标杆在把杠杆转成“努力需要放大器”的行动判断。
- 可复用点：强冲突开头、具体工作案例、系列标识、进度线、烧录字幕、独立封面。
- 仍可提升点：配音真人感、标题 A/B、发布后基于完播率压缩节奏。
